# 서비스 구성 제안 - 프로젝트 개요

## 📋 프로젝트 소개

**데이터 통합 플랫폼 기반 지능형 IoT 관리 솔루션**의 프레젠테이션 및 대시보드 애플리케이션입니다.

이 프로젝트는 분산된 IoT 데이터를 통합하고, 실시간 모니터링, 원격 제어, AI 기반 분석을 통해 지능형 IoT 관리 솔루션을 제공하는 시스템의 설계 및 제안 자료를 웹 애플리케이션으로 구현한 것입니다.

### 주요 특징

- **3가지 모드 제공**: 프레젠테이션, 대시보드, 설계 문서 모드
- **실시간 데이터 시각화**: 차트 및 대시보드를 통한 데이터 분석
- **Mermaid 다이어그램**: 프로세스 플로우를 시각적으로 표현
- **반응형 디자인**: 다양한 화면 크기에 최적화
- **URL 기반 네비게이션**: 해시 기반 라우팅으로 직접 링크 공유 가능

## 🚀 시작하기

### 사전 요구사항

- **Node.js**: 18.x 이상
- **npm**: 9.x 이상 (또는 yarn, pnpm)

### 설치

```bash
# 의존성 설치
npm install
```

### 개발 서버 실행

```bash
# 개발 서버 시작 (Vite)
npm run dev
```

개발 서버는 기본적으로 `http://localhost:3000`에서 실행됩니다.

**주요 기능**:
- Hot Module Replacement (HMR): 코드 변경 시 즉시 반영
- 빠른 빌드 속도: Vite의 최적화된 빌드 시스템
- TypeScript 지원: 타입 안정성 보장

### 프로덕션 빌드

```bash
# 프로덕션 빌드
npm run build

# 빌드 결과 미리보기
npm run preview
```

빌드 결과물은 `dist/` 디렉토리에 생성됩니다.

## 📱 애플리케이션 모드

이 애플리케이션은 **3가지 모드**를 제공하여 다양한 사용자 그룹의 needs를 충족합니다.

### 3가지 모드의 관계

```
프레젠테이션 모드 (비즈니스 관점 - 개요)
    ↓ 구체화
기술 상세 모드 (기술 관점 - 상세 분석)
    ↓ 구체화
설계 문서 모드 (상세 기술 문서 - 최종 구체화)
```

**핵심 개념**: 
- **프레젠테이션 모드**는 비즈니스 제안용으로 핵심 메시지와 개요를 제시
- **기술 상세 모드**는 프레젠테이션의 기술적 상세 분석 버전
- **설계 문서 모드**는 프레젠테이션과 기술 상세 모드의 **구체화된 상세 버전**

### 1. 프레젠테이션 모드 (Presentation Mode) - 기본 모드

**목적**: 고객 제안 및 비즈니스 검토용 프레젠테이션 - **개요 및 핵심 메시지 중심**

**주요 기능**:
- 7개 슬라이드로 구성된 프레젠테이션
  - 제목 슬라이드
  - 현재 문제점 분석 (→ 기술 상세 모드 Frame 1에서 상세 분석)
  - 솔루션 개요 (→ 기술 상세 모드 Frame 2, 3, 6에서 상세 분석)
  - 시스템 아키텍처 (→ 기술 상세 모드 Frame 3에서 상세)
  - 핵심 기능 및 이점 (→ 기술 상세 모드 Frame 4, 5에서 상세)
  - 구현 로드맵 (→ 기술 상세 모드 Frame 7에서 상세)
  - ROI & 비즈니스 임팩트 (→ 설계 문서에서 비용 분석 상세)
- 키보드/마우스 네비게이션 지원
- 슬라이드 인디케이터 및 진행률 표시
- 전체화면 모드 지원

**접속 방법**: 
- 기본 접속: `http://localhost:3000` (자동으로 프레젠테이션 모드)
- URL 해시: `#presentation/{slide-id}` (예: `#presentation/title`)

**대상 사용자**: 고객사, 경영진, 의사결정자

**설계 문서와의 관계**: 각 슬라이드의 내용이 설계 문서에서 기술적으로 상세하게 설명됨

### 2. 기술 상세 모드 (Technical Detail Mode) - 기술팀용

**명칭 변경 제안**: "대시보드 모드" → "기술 상세 모드" (또는 "상세 분석 모드")

**목적**: 상세 분석 및 기술 검토용 - **프레젠테이션 모드의 기술적 상세 버전**

**주요 기능**:
- 7개 프레임으로 구성된 상세 분석
  - 데이터 문제 정의 (프레젠테이션 Slide 2의 기술적 상세)
  - 데이터 표준화 (프레젠테이션 Slide 3의 데이터 표준화 부분 상세)
  - 데이터 통합 플랫폼 (프레젠테이션 Slide 4의 데이터 통합 부분 상세)
  - 실시간 모니터링 (프레젠테이션 Slide 5의 모니터링 부분 상세)
  - 원격 제어 및 OTA (프레젠테이션 Slide 5의 원격 제어 부분 상세)
  - 지능형 분석 (프레젠테이션 Slide 3의 AI 기반 분석 부분 상세)
  - 데이터 활용 확장 (프레젠테이션 Slide 6의 확장 계획 상세)
- 실시간 데이터 시각화 (차트, 그래프)
- KPI 지표 카드
- 상세 기술 정보 제공

**접속 방법**: 
- 화면 우측 상단 "대시보드" 버튼 클릭
- URL 해시: `#dashboard/{frame-id}` (예: `#dashboard/problem`)

**대상 사용자**: 기술팀, 아키텍트, 개발팀

**설계 문서와의 관계**: 각 프레임의 내용이 설계 문서에서 더욱 상세하게 다이어그램과 프로세스로 설명됨

### 3. 설계 문서 모드 (Design Document Mode) - 구체화 버전

**핵심 개념**: 설계 문서 모드는 **프레젠테이션 모드와 기술 상세 모드의 구체화된 상세 버전**입니다.

**프레젠테이션 모드의 구체화**:
- 각 슬라이드의 내용을 기술적으로 상세히 설명
- 슬라이드에서 제시한 개념과 솔루션이 구체적인 프로세스와 다이어그램으로 확장

**기술 상세 모드의 구체화**:
- 각 프레임의 내용을 기술 문서로 상세화
- 프레임에서 보여준 내용의 기술적 배경과 구현 방법을 상세히 설명

**주요 기능**:
- 마크다운 문서 파싱 및 렌더링
- Mermaid 다이어그램 렌더링 (확대/축소, 전체화면 지원)
- 섹션별 네비게이션
- 반응형 레이아웃
- 문서 목록:
  - 프로세스 플로우 (PROCESS_FLOW.md) - 전체 시스템 프로세스 상세
  - 프로젝트 분석 (PROJECT_ANALYSIS.md) - 프로젝트 상세 분석
  - 프로젝트 개요 (README.md) - 프로젝트 개요 및 가이드

**접속 방법**: 
- 화면 우측 상단 "설계 문서" 버튼 클릭
- URL 해시: `#docs/{doc-id}/{section-id}` (예: `#docs/process-flow`)

**대상 사용자**: 
- 모든 이해관계자
- 프레젠테이션을 본 후 상세 내용을 확인하고 싶은 사용자
- 기술 상세 모드를 본 후 기술적 세부사항을 확인하고 싶은 개발팀

## 🛠️ 기술 스택

### 핵심 기술

- **빌드 도구**: Vite 6.3.5
  - 빠른 개발 서버 및 최적화된 빌드
  - Hot Module Replacement (HMR) 지원
- **프레임워크**: React 18.3.1 + TypeScript
  - 컴포넌트 기반 아키텍처
  - 타입 안정성 보장

### UI 및 스타일링

- **CSS 프레임워크**: Tailwind CSS
  - 유틸리티 기반 스타일링
  - 반응형 디자인 지원
- **UI 컴포넌트**: Radix UI + shadcn/ui
  - 접근성 기반 컴포넌트 (20개 이상)
  - 일관된 디자인 시스템
- **아이콘**: Lucide React
  - 1000개 이상의 아이콘 제공

### 데이터 시각화 및 문서

- **차트 라이브러리**: Recharts 2.15.2
  - 다양한 차트 타입 지원 (Line, Bar, Pie 등)
  - 인터랙티브 차트 기능
- **다이어그램**: Mermaid.js
  - 프로세스 플로우 시각화
  - 확대/축소 및 전체화면 지원
- **마크다운**: react-markdown + remark-gfm
  - GitHub Flavored Markdown 지원
  - 코드 하이라이팅

### 애니메이션 및 기타

- **애니메이션**: Motion (Framer Motion fork)
  - 부드러운 전환 효과
  - 성능 최적화된 애니메이션
- **폼 관리**: react-hook-form
- **토스트 알림**: Sonner
- **테마**: next-themes (다크모드 지원)

## 개발 가이드

### 컴포넌트 추가
새로운 슬라이드나 프레임을 추가하려면:
1. `src/apps/presentation/slides/` 또는 `src/components/`에 컴포넌트 생성
2. 해당 앱의 메인 컴포넌트에 등록

### 스타일 커스터마이징
- 전역 스타일: `src/styles/globals.css`
- Tailwind 설정: `tailwind.config.js` (필요시)
- 컴포넌트별 스타일: 각 컴포넌트 내부에서 Tailwind 클래스 사용

### 모드 전환 로직
`src/App.tsx`에서 모드 전환 로직을 관리합니다:
- URL 파라미터 기반 모드 감지
- 상태 기반 모드 전환
- 히스토리 API를 이용한 URL 업데이트

## 문제 해결

### 포트 충돌
기본 포트 3000이 사용 중인 경우, `vite.config.ts`에서 포트를 변경할 수 있습니다.

### 빌드 오류
의존성 문제가 발생하면:
```bash
rm -rf node_modules package-lock.json
npm install
```
